# Instructional Designer - AI Agent Template
## Accessibility Implementation

### Success Definition (Measurable)
- **Goal:** Achieve a fully accessible learning environment where all content is compliant with WCAG 2.1 AA standards.
- **Success Metrics:**
  - **Accessibility Score:** ≥ 100% compliance on automated accessibility scans.
  - **User Feedback:** ≥ 90% positive feedback from users in usability tests, especially those using assistive technologies.
  - **Compliance Verification:** Pass all manual checks and third-party audits for WCAG 2.1 AA standards.

### Critical Knowledge Areas

**Research Agent Deployment:** Launch 10 parallel sub-agents to research latest accessibility practices.

**Topic 1: Understanding WCAG Standards**
- Research Focus: [2024-2025] versions of WCAG, key principles (Perceivable, Operable, Understandable, Robust).
- Target Sources: W3C WCAG documentation, industry blogs.
- Deliverable: Key principles and guidelines with examples.

**Topic 2: Instructional Design Best Practices for Accessibility**
- Research Focus: [2024] strategies for designing accessible content, inclusive learning experiences.
- Target Sources: eLearning Journal, CAST publications.
- Deliverable: Guidelines for creating accessible lesson plans and materials.

**Topic 3: Tools for Creating Accessible Content**
- Research Focus: [Best free tools in 2024] for authoring accessible PDFs, videos, interactive elements (e.g., Articulate Storyline, Adobe Captivate).
- Target Sources: Product documentation, user reviews.
- Deliverable: Tool comparison matrix highlighting accessibility features.

**Topic 4: Screen Reader Compatibility**
- Research Focus: [Testing methods and tools] for ensuring screen readers compatibility with instructional materials.
- Target Sources: NVDA, JAWS documentation; browser extensions.
- Deliverable: Checklist of screen reader tests passed.

**Topic 5: Captions and Transcripts**
- Research Focus: [Best practices for creating high-quality captions] and transcripts for video content.
- Target Sources: TEITAC guidelines, captioning tools reviews.
- Deliverable: Sample caption file and transcript template.

**Topic 6: Alternative Text (Alt Text)**
- Research Focus: How to write effective alt text for images in e-learning modules.
- Target Sources: Alt text writing guides, accessibility checkers.
- Deliverable: Guidelines document with sample alt texts.

**Topic 7: Color Contrast and Visual Accessibility**
- Research Focus: [2024] recommendations for ensuring sufficient color contrast and designing visually accessible interfaces.
- Target Sources: Color Contrast Checker tools, A11y project guidelines.
- Deliverable: Checklist of color contrast ratios met.

**Topic 8: Keyboard Navigation**
- Research Focus: Ensuring all interactive elements are navigable using keyboard alone.
- Target Sources: Keyboard navigation tutorials, accessibility testing tools.
- Deliverable: Manual test report demonstrating full keyboard functionality.

**Topic 9: Inclusive Design Principles**
- Research Focus: [Emerging trends in inclusive design] for diverse learner populations (e.g., neurodiversity, cultural inclusivity).
- Target Sources: Inclusive Design Journal, community forums.
- Deliverable: Set of inclusive design principles and case studies.

**Topic 10: Legal Compliance and Reporting**
- Research Focus: Understanding [2024] legal requirements for accessibility in educational content (e.g., Section 508, GDPR).
- Target Sources: Legal databases, regulatory bodies.
- Deliverable: Compliance checklist with reporting templates.

### Execution Steps

**STEP 1: Assess Current Content Accessibility**
- **Action:** Conduct an initial audit using automated tools like WAVE or Axe to identify non-accessible content.
- **Tools Needed:** WAVE (free), Axe (optional premium), NVDA/JAWS for screen reader testing.
- **Success Criteria:** Documented list of all accessible issues found.
- **Common Pitfalls:** Over-reliance on automation; missing critical visual elements.
- **Time Estimate:** 1 week

**STEP 2: Prioritize and Plan Remediation**
- **Action:** Categorize issues by severity (critical, high, medium, low) and create a remediation backlog.
- **Tools Needed:** JIRA or Trello for issue tracking.
- **Success Criteria:** All current accessibility issues logged in the backlog with priority levels assigned.
- **Common Pitfalls:** Delayed prioritization leading to incomplete fixes.
- **Time Estimate:** 3 days

**STEP 3: Create Accessible Learning Materials**
- **Action:** Redesign or create new instructional materials ensuring all content is accessible (e.g., alt text, captions).
- **Tools Needed:** Adobe Captivate (free trial), Articulate Storyline (optional premium), captioning software like Rev.com.
- **Success Criteria:** All new/updated learning assets pass automated accessibility scans and manual review.
- **Common Pitfalls:** Inconsistent use of alt text or captions.
- **Time Estimate:** 4 weeks

**STEP 4: Implement Interactive Elements**
- **Action:** Ensure interactive elements (quizzes, simulations) are accessible with keyboard navigation and screen readers.
- **Tools Needed:** Articulate Rise (free), Adobe Captivate (optional premium).
- **Success Criteria:** All interactive components pass accessibility testing tools like Axe or Lighthouse.
- **Common Pitfalls:** Hidden focus states; non-navigable elements.
- **Time Estimate:** 2 weeks

**STEP 5: Conduct User Testing**
- **Action:** Recruit users with diverse needs to test the learning materials and provide feedback on accessibility.
- **Tools Needed:** SurveyMonkey (free), usability testing software like Lookback.io.
- **Success Criteria:** ≥ 90% of testers report no significant accessibility issues; documented feedback incorporated.
- **Common Pitfalls:** Lack of diverse tester group leading to blind spots.
- **Time Estimate:** 1 week

**STEP 6: Final Compliance Verification**
- **Action:** Perform a final audit and manual testing for all accessible features, including screen readers and keyboard navigation.
- **Tools Needed:** NVDA/JAWS (free), automated accessibility scanners like Axe.
- **Success Criteria:** All materials pass full compliance check with no critical issues remaining.
- **Common Pitfalls:** Skipping final review due to time constraints.
- **Time Estimate:** 3 days

**STEP 7: Documentation and Training**
- **Action:** Document all changes, create training materials for staff on accessibility best practices.
- **Tools Needed:** Confluence (free), PowerPoint/Google Slides (optional premium).
- **Success Criteria:** All team members have completed accessibility training; documentation is stored in shared drive.
- **Common Pitfalls:** Incomplete or missing documentation.
- **Time Estimate:** 1 week

**STEP 8: Maintenance and Ongoing Improvement**
- **Action:** Set up a schedule for regular audits, updates, and user feedback collection to ensure long-term accessibility.
- **Tools Needed:** Automated testing tools like Axe; project management software (e.g., Asana).
- **Success Criteria:** Regular maintenance is logged in the schedule; issues are tracked and resolved within 30 days.
- **Common Pitfalls:** Lack of ongoing commitment leading to regression over time.
- **Time Estimate:** Ongoing

### Troubleshooting Common Issues

**Issue 1:** Content Not Scanning Accessible by Automated Tools
- **Solution:** Manually test using screen readers and keyboard navigation. Address any missing alt text or captions.

**Issue 2:** High Priority Accessibility Issues Remain Unresolved
- **Solution:** Prioritize fixes based on user impact; involve stakeholders in review process.

**Issue 3:** Team Not Adhering to Best Practices Post-Training
- **Solution:** Conduct regular refresher training sessions and audits to reinforce best practices.

**Issue 4:** Legal Compliance Concerns Identified
- **Solution:** Consult with legal experts to address specific compliance requirements; update documentation accordingly.
